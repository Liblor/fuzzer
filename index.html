Overview

This project is a Python, mutation based file fuzzer that uses PyDBG to monitor for signals of interest.

Architecture

The architecture is a multiprocessing fuzzer, following a producer/consumer model. For every mutation that the mutator applies to the original file, add a new job to the In_queue. One of the n processes will get the job from the queue, and using PyDBG, execute the application with the mutated file and monitor for any signals of interest. After the run finishes, or a timeout occurs, the process executing the job will put any output in the Out_queue.
